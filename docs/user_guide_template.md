# Руководство пользователя

## 1. Назначение системы

Информационно-аналитическая система сравнительного анализа объектов предназначена для загрузки табличных данных, их подготовки к сравнению, настройки критериев анализа и получения итогового результата в виде рейтинга объектов или списка аналогов относительно выбранного целевого объекта.

Система позволяет:

- загружать пользовательские датасеты;
- просматривать структуру и качество данных;
- настраивать подготовку признаков;
- выполнять ранжирование объектов;
- искать аналоги по выбранным критериям;
- оценивать стоимость целевого объекта по аналогам;
- сохранять историю расчетов и формировать отчеты.

## 2. Начало работы

Для начала работы откройте web-интерфейс системы в браузере.

Если требуется сохранение истории, работа с проектами или доступ к административным функциям, выполните вход в учетную запись. Если авторизация не обязательна для вашего сценария, базовый анализ можно выполнять и без входа.

- распознает формат;
- строит предпросмотр;
- определяет колонки;
- вычисляет базовые характеристики полей;

## 7. Подготовка данных

Этап подготовки данных предназначен для настройки признаков перед запуском аналитического расчета. На этом этапе пользователь управляет тем, как система интерпретирует и преобразует данные.

Подготовка обычно включает:

- настройку типов данных;
- включение и исключение признаков;
- выбор полей, формирующих название объекта;
- настройку энкодинга категорий;
- обработку пропусков;
- обработку выбросов;
- масштабирование числовых признаков.

## 8. Настройка типов данных

Во вкладке типов данных пользователь может:

- выбрать тип признака;
- исключить лишние поля из анализа;
- отметить поля, используемые в названии объекта;
- задать точность для дробных значений;
- выбрать формат даты;
- выбрать единицы измерения для числовых полей, если они были распознаны системой.

Рекомендуется исключать признаки, которые не несут аналитической ценности, например:

- внутренние идентификаторы;
- служебные коды;
- уникальные текстовые строки, не подходящие для сравнения.

## 15. Режим поиска аналогов

В режиме поиска аналогов пользователь:

1. Выбирает целевой объект.
2. Задает критерии, по которым определяется близость.
3. При необходимости ограничивает поиск по радиусу.
4. Запускает расчет аналогов.

Целевой объект выбирается через поле поиска, что позволяет работать не только с первыми строками предпросмотра, а со всем датасетом.

## 24. Рекомендации по использованию

Для получения корректного результата рекомендуется:

- внимательно проверять структуру загруженного датасета;
- исключать из анализа поля, не несущие аналитического смысла;
- задавать осмысленные веса критериев;
- не использовать стоимость как критерий близости, если она применяется для оценки по аналогам;
- проверять корректность ручного энкодинга категорий;
- использовать поиск аналогов с радиусом, если важна пространственная близость объектов.

## 25. Типичные ошибки пользователя

Наиболее распространенные затруднения:

- загрузка файла с неподходящей структурой;
- включение в анализ уникальных идентификаторов;
- слишком большой вес у второстепенного признака;
- неверный выбор типа данных;
- использование поля стоимости одновременно и как критерия, и как поля оценки.

При возникновении таких ситуаций рекомендуется вернуться к этапу подготовки данных и критериям, скорректировать настройки и повторить расчет.

## 26. Заключение

Система предоставляет полный цикл сравнительного анализа объектов: от загрузки и подготовки данных до получения интерпретируемого результата. Руководство пользователя должно использоваться как пошаговая инструкция по работе с интерфейсом и аналитическими возможностями приложения.
