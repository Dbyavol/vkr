# Comparative Analysis Service

Изолированный микросервис сравнительного анализа объектов для микросервисной архитектуры.

## Что принимает сервис

Сервис принимает один JSON-документ:

- `dataset.objects` — массив объектов анализа
- `criteria` — список критериев с весами и правилами нормализации
- `target_object_id` — опциональный целевой объект для оценки похожести
- `top_n` — сколько лучших объектов вернуть

Каждый объект содержит:

- `id`
- `title`
- `attributes` — произвольный словарь признаков

Такой формат универсален: остальные микросервисы могут подготовить датасет из БД, файлового хранилища или внешних источников и передать его в аналитический сервис без привязки к предметной области.

## Что возвращает сервис

- итоговый рейтинг объектов
- интегральный `score`
- похожесть на целевой объект
- вклад каждого критерия
- краткое текстовое объяснение
- служебную сводку по нормализации

## Поддерживаемые типы критериев

- `numeric`
- `categorical`
- `binary`

## Поддерживаемые направления

- `maximize`
- `minimize`
- `target`

## Локальный запуск

```bash
pip install -r requirements.txt
uvicorn app.main:app --reload --port 8080
```

## Пример запроса

```bash
curl -X POST http://localhost:8080/api/v1/analysis/run ^
  -H "Content-Type: application/json" ^
  --data-binary "@example-request.json"
```

## Роль в общей системе

Этот сервис должен быть центральным аналитическим микросервисом и вызываться другими компонентами:

- сервисом управления объектами
- сервисом импорта и подготовки данных
- frontend-приложением через API gateway

В следующем шаге его можно расширить:

- сохранением результатов в отдельное хранилище
- шаблонами анализа
- чувствительностью к изменению весов
- асинхронными задачами для больших датасетов
