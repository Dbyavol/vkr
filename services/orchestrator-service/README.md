# Orchestrator Service

Единая точка входа для пользовательского сценария:

1. принять файл
2. построить preview
3. применить предобработку
4. выполнить сравнительный анализ
5. вернуть агрегированный результат для интерфейса

## Endpoint

- `POST /api/v1/pipeline/run`

Форма запроса:

- `file` — CSV/XLSX/JSON
- `config_json` — JSON-строка с настройками полей и критериев

## Назначение

Этот сервис нужен, чтобы frontend не вызывал каждый микросервис отдельно. Сейчас он выполняет локальную orchestration-логику и повторяет контракты ранее созданных сервисов. Дальше его можно переключить на реальные HTTP-вызовы в:

- `import-service`
- `preprocessing-service`
- `comparative-analysis-service`
- `storage-service`
