# Preprocessing Service

Изолированный микросервис предобработки датасета перед передачей в сервис сравнительного анализа.

## Назначение

Сервис принимает сырой датасет и конфигурацию pipeline, после чего возвращает:

- очищенные строки датасета
- нормализованные числовые признаки
- закодированные категориальные и бинарные признаки
- отчет по каждому полю
- сводку по удаленным дублям, пропускам и выбросам

## Формат входа

- `dataset.rows[].id` — идентификатор строки
- `dataset.rows[].values` — словарь признаков
- `fields[]` — правила обработки по каждому полю
- `options` — глобальные настройки pipeline

## Что умеет сервис

### Обработка пропусков

- `none`
- `drop_row`
- `mean`
- `median`
- `mode`
- `constant`

### Обработка выбросов

- `none`
- `iqr_remove`
- `iqr_clip`
- `zscore_remove`
- `zscore_clip`

### Нормализация

- `none`
- `minmax`
- `zscore`
- `robust`
- `log_minmax`

### Энкодинг

- `none`
- `one_hot`
- `ordinal`
- `binary_map`

## Локальный запуск

```bash
pip install -r requirements.txt
uvicorn app.main:app --reload --port 8090
```

## Пример вызова

```bash
curl -X POST http://localhost:8090/api/v1/preprocessing/run ^
  -H "Content-Type: application/json" ^
  --data-binary "@example-request.json"
```

## Связь с аналитическим сервисом

Рекомендуемый поток в системе:

1. Сервис импорта загружает исходный файл.
2. Сервис предобработки очищает и преобразует датасет.
3. Сервис сравнительного анализа получает уже подготовленные признаки.

Для связки с аналитическим сервисом достаточно преобразовать результат `dataset[].values` в формат `attributes` и передать его дальше.
